\-0509

seata启动报错：【Caused by: com.alibaba.nacos.api.exception.NacosException: failed to req API:/nacos/v1/ns/instance after all servers(\[localhost:8848]) tried: java.net.ConnectException: Connection refused: no further information】
---原因：【seata-server\\conf\\application.yml】除了seata的config配置nacos地址，register也要配！否则会默认8848

\---解决：为registry也添加

\---附注：可以不填账密，因为nacos.core.auth.enabled 默认 =false



seata server启动后，Store Mode / 依旧是 file

&#x09;原因和解决方案：虽然nacos 的seataServerTest.properties文件名和seata的yml配置的data-id匹配，但是nacos中，配置格式勾选的是Text！且nacos.group也不匹配

&#x09;解决方案：修改配置格式为Properties，group也修正为匹配

seata server启动报错：The driver {com.mysql.cj.jdbc.Driver} cannot be found in the path ... seata-server/lib/jdbc

&#x09;原因：\\seata-server\\lib\\jdbc\\目录需要jdbc.jar

&#x09;解决：从本地maven仓库复制一个【mysql-connector-j-9.5.0.jar】过去

order起不来报错找不到Seata server

&#x09;原因：测试环境参考的命名空间有两个看起来配置都是11个，但是我拷贝的seataServerTest里面缺了一个配置

&#x09;解决：补上【service.vgroupMapping.default\_tx\_group=default】即可

&#x09;补充：这个虚拟集群名称是必须的，默认就叫default

&#x09; 昏头错误：把web微服务本地resources里的application-local.yml里spring.cloud.nacos.config.group改为了seata\_group，这个别乱写;

&#x09;若还不行，就直接在web微服务的优先级yml里的seata.service.default加上grouplist: 127.0.0.1:8091 来强制直连
