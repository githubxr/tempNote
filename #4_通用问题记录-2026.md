##### 0518~23

- 0522
- environment variables 配置的nacos 命名空间id无效（地址有效...)
  - 解决（名称改为这两个（discover发现和config配置都得配））：
    - SPRING_CLOUD_NACOS_DISCOVERY_NAMESPACE
    - SPRING_CLOUD_NACOS_CONFIG_NAMESPACE

- nacos配置无法覆盖resources的yaml配置

  - 解决：

    - 1：

    ```
    #添加以开启nacos覆盖
    pring:
      cloud:
        nacos:
          config:
            override-none: false   # 允许覆盖本地配置【必须加】
            override-system-properties: false
            refresh-enabled: true  # 自动刷新
    ```

    - 2：jvm参数覆盖【xx.yy.zz=aaa】（优先级最高）



- JVM环境变量启动参数中，指定nacos命名空间无效

  - 描述（两者均无效）：

    - spring.cloud.nacos.config.namespace
    - spring.cloud.nacos.discover.namespace

  - 解决（替换为这俩）：

    - SPRING_CLOUD_NACOS_DISCOVERY_NAMESPACE
    - SPRING_CLOUD_NACOS_CONFIG_NAMESPACE

  - 解释（nacos缺陷，启动读取顺序问题）：

    - `Dkey=value` 这种 JVM 参数

      是Spring 上下文启动后才会加载生效 → 太晚了

    - `SPRING_CLOUD_**` 这种环境变量格式

      是操作系统级环境变量，Spring 一启动立刻就能读到 → 刚好来得及

  - 拓展：

    - 所有yaml配置都自动有下划线拼接的变量名可用！
    - 例如a.b.c.d，就有A_B_C_D可用！
